西安分公司线上bug

Posted on 2025-05-15
  • Kuk
Home

BUG 描述：

在修改人员信息的岗位时，出现报错信息：” 该岗位 id 非隶属于该组织”。

报错环境：西安内网。

排查思路：

  1. 查看本地代码，尝试复现 BUG。
  2. 抓取一次请求，分析岗位 ID 和组织 ID 是否与数据库中的数据一致。
  3. 由于无法进行断点调试，通过在代码中添加打印语句输出到控制台进行排查。

实际排查过程：

  1. 查阅西安内网日志，查看报错原因和位置。
  2. 检查本地代码是否存在逻辑错误。
  3. 尝试在本地环境复现 BUG。
  4. 在西安内网抓取一次请求，分析传递的组织 ID 和岗位 ID 是否正确。检查前端传参是否有误。
  5. 使用 Compass 检查数据是否存在且有效。
  6. 分析可能与问题相关的代码，例如岗位是否与组织正确关联，岗位接口是否正常。
  7. 在代码中添加打印语句，将组织 ID 和岗位 ID 输出到日志中。将代码打包上传至西安内网。
  8. 在日志中检查数据，看是否存在错误。

最终在日志中发现，在查询岗位时没有使用前端传递的组织 ID，而是使用了人员修改前的组织 ID 进行岗位查询。

最后发现该 BUG 的触发条件是，如果在同一组织下进行岗位调整，则不会触发该 BUG，但是当进行跨组织调岗时才会触发。

修正描述：根据排查结果，发现在岗位查询时未使用前端传递的组织 ID，而是使用了人员修改前的组织 ID 进行查询。因此，针对岗位查询时的组织 ID 应使用前端传参，以解决该 BUG。

个人总结：

  1. 修复 bug 时间总供用了 4 个小时
    1. 主要原因是查看西安内网数据是只能通过 todesk 工具连接 ，中间耗费了 2 个小时
    2. 排查错误时同时看本地是否能复现 bug 在本地修改的时间成本 会小很多
