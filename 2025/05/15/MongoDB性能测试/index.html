# 二、测试环节

  1. # mongoimport 工具分别向单机和分片集群导入

    1. 磁盘性能未能完全利用
    2. 由于工具闭源所以写 java 程序导入

  2. # 使用自己写的 java 程序导入

    1. 导入更慢、磁盘性能未能完全利用 — 经监控发现 gc 频繁

    2. 调整 JVM 参数策略单机完成磁盘完全利用（1200 万条数据 - 2.5 分钟 / 写 (140m/s) 读 (90m/s)）

    3. 分片模式使用上述策略未见效果–分析 1、mongos 读异常 shard（10 M/s）

    4. 更新 MongoDB 版本未见效果、使用 import 导入工具多线程导入读 (40 M/s)

    5. 修改代码对 insertMany () 的参数做调整有明显提高（最高可达写 100M/s）

      1. 异步写入(只发送写入的请求不做验证) 	  -writeConcern			
是否排序(是否按照顺序写)			 -ordered(影响显著)
是否校验规则(是否有自定义导入规则校验) -bypassDocumentValidation

      2. 此时由于读的原因 (封顶 150M/s) 已经写的速度已经开始收到影响（2 片一共 160M/s）（3 片一共 190M/s）

    6. 读流量的增加

# 三、测试结果
