# 3、前端 — 实现核心部分

sso - 调用

  • 需要全局拦截所有路径带有 SSO 的请求
  • 模拟登录功能
  • 重定向到真正的访问的页面

# 4、后端 — 实现核心部分

sso - 调用

  • 第三方会提供一个 access_key
  • 使用 access_key 以及 encryAccount 请求第三方提供的解密接口
  • 对解密完的数据 进行校验 该用户存在 返回给前端登录所需要的信息

sso - 集成

  • 注册该功能时 需要考虑加密算法的使用 （目前 AES）
  • 提供解密的接口 设计响应体
  • 提供一个重定向的接口 用于实施人员部署的到前台
