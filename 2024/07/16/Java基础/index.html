# java 基础

# 第四章 - 面向对象

# 1、抽象方法

# def

1、所谓抽象就是：没有方法体 的 方法

2、用 abstract 修饰的类叫抽象类 abstract 修饰的方法叫抽象方法

# 使用背景

1、父类 方法 不确定要实现为什么，换句话说就是为了后期去重写的（一般子类会去重写）

# 使用方法

用 abstract 关键字 修饰

# Detail

1、抽象类不能被实例化

2、抽象类不一定包括抽象方法（抽象类，可以没有抽象方法，而且可以有实现的方法）

3、一个类一旦包含了 abstract 方法，则该类必须声明为 abstract

4、abstract 只能修饰类和方法， 不能修饰其他方法。

5、抽象类 == 类 本质是类（抽象方法，构造器，静态属性）

6、☆☆☆抽象类不能有主体

7、如果一个类继承了抽象类，则他必须实现抽象类的所有抽象方法，除非他自己也声明为 abstract 类

8、☆☆☆☆☆☆抽象方法不能被 private,final,static 修饰符修饰，因为修饰符都涉及到重写，与之相违背

static 静态方法可以被继承但是不能被重写

# 2、代码块

# Def

代码化块又叫初始化块，属于类中的成员（即是类的一部分），类似于方法，将逻辑语句封装在方法体中，通过 {} 包围

不用通过对象或类显示调用，而是类加载时，或创建对象时隐式调用

# 基本语法

修饰符 {

注意 1、修饰符可选，只能用 static

2、分为两类一种被 static 修饰叫静态代码块，另一种被称为普通代码块

3、；可以省略可以写

4、逻辑语句任意

# Detail☆☆☆☆

1、代码块的调用的顺序优先于构造器

2、静态代码块，作用对类进行初始化，而且随着类加载而执行，只会执行一次；

3、普通代码块，创建实例时，会被隐式调用。被创建一次，调用一次。如果只是使用类的静态成员时，普通代码块不会被加载

4、☆☆☆☆☆☆类什么时候被加载

①创建对象实例时（new）

②创建子类对象实例时，父类也会被加载

③使用静态成员时

5、如果有继承关系泽；先加载父类的代码块，再加载子类的代码块（因为父类构造器先加载，而代码块的调用的顺序优先于构造器 ）

6、☆☆☆☆☆☆创建一个对象时，在一个类 调用的顺序是

①调用静态代码块 和 静态属性初始化（静态代码块和静态属性初始化 调用优先级是一样、、如果有多个则按顺序执行）

②调用普通代码块 和 普通属性初始化

③调用构造方法

7、☆☆☆☆构造器最前面 隐含了 super（）和调用普代码块，

8、☆☆☆☆☆☆创建一个子类时（继承关系），静态代码块，静态属性初始化，普通代码块，普通属性初始化，构造器的顺序

①父 静代码块和静属性 （优先级一样，按顺序执行）

②子 静代码块和静属性 （优先级一样，按顺序执行）

③父 普通码块和普通属性 （优先级一样，按顺序执行）

④父 构造器

⑤子 普通码块和普通属性 （优先级一样，按顺序执行）

⑥子 构造器

9、静态代码块只能调用静态成员，普通代码块可以调用任意成员

# 3、单例设计模式

# 一、单例（单个实例）

1、所谓的单例设计模式，就是采取一定的方法保证这个软件系统中，对某个类只能存在一个实例对象，并且该类只提供一个取得其对象的方法

2、单列模式有两种方式：①饿汉式

饿汉式缺点

造成资源浪费，因为对象会被直接创建

②懒汉式

只有用户使用时才会创建对象

# 4、动态绑定机制（☆☆☆☆☆）

1、当调用对象方法的时候，该方法会和该对象的 内存地址（运行类型） 绑定

2、当调用对象属性时，没有动态绑定计机制，哪里声明 哪里使用

# 5、断点调试

# 一、介绍

用来寻找错误的方法，注意在断点调试过程中 是运行状态，是以对象的运行类型来执行的

# 二、快捷键

F7 ： 跳入 跳入方法内

F8 ： 跳过 逐行执行代码

Shift + F8 ：跳出 跳出方法

F9 ： 执行到下一个断点

alt+shift+f7：强制跳入

# 6、多态

# 一、Def

方法或对象具有多种形态，是面向对象的第三大特征

# 二、体现

# 方法的多态

方法的重载与重写

# 对象的多态

一个对象的编译类型和运行类型可以不一致

编译类型在定义的时候就确定了，不能改变

运行类型可以改变

编译类型看 = 左边 运行类型看 = 右边

# 多态的向上和向下
# 向上转型

举例：

本质：父类的引用指向子类

语法：引用名 = new 子类类型 ()；

特点：可以调用父类中所有成员（需要遵守访问规则）

不能调用子类特有成员

最终运行效果看子类的实现

# 向下转型

举例

用途：调用子类的特有方法（因为向下转型后 编译类型和运行类型一致所以可调用特有的方法，换句话说向下转型后与 Dog dog = new Dog (); 相同）

本质：将父类的引用转回子类引用

语法：子类类型 引用名 = (子 ± 型) 父类引用；

特点：只能强转父类的引用，不能强转父类的对象

要求 父类的引用 必须 是指向的是当前的目标类型 的对象（可理解为 之前就是作为引用的）

当向下转型后就可以调用子类中的所有成员

2、当调用对象属性时，没有动态绑定计机制，哪里声明 哪里使用

# 4.1 多态数组

4.1.1 定义

# 4.2 多态参数

4.2.1 定义

# 7、继承和接口关系

# 一、继承接口解决的问题不一样

1、继承的价值主要用在：解决代码的服用性和可维护性

2、接口的价值主要在于：设计各种规范，让其类实现这些方法

# 二、接口比继承更加灵活

1、继承需要满足 is - a 的关系 ，而接口只需要满足 like - a 的关系即可

# 三、接口在一定程度上可以实现代码的解耦

接口的规范性 + 动态绑定机制

# 8、接口

# 一、基本接口

1、给出一些没有实现的方法，封装到一起

# 二、语法

interface 接口名 {

// 属性

// 方法

}

class 类名 implements 接口 {

自己属性

自己方法

必须实现的接口的抽象方法

}

[注意] jdk 7 前 接口类的所有方法都没有方法体，即是抽象的方法体

[注意] jdk 8 后 接口可以有静态方法，默认方法，也就是说接口中可以有方法的具体实现

# 三、细节

1、接口不能实例化

2、接口中所有的方法都是 public 的，可以不用 abstract 修饰

3、一个普通类实现接口，就必须将该接口的所有方法都实现。

4、抽象类实现接口，可以不用实现接口方法

5、☆一个类可以实现多个接口

6、接口中的属性只能是 final，而且是 public static final 修饰符

7、接口的访问形式：接口名。属性名

8、一个接口不能继承他的类，但是可以继承多个别的接口

9、接口的修饰符 只能是 public 和默认，这点和类的修饰符是一样的

# 9、类方法和类变量

# 一、类变量

# 1、实例变量和类变量的区别

1.1 实例变量在创建对象是分配空间，类变量在类加载后就会被分配空间

1.2 所有对象共享类变量，对同一个类所有对象共享（）

1.3 实例变量通过对象访问 不能通过类名访问，类变量可以通过类名访问也可以通过对象访问（因为类创建了后类变量一定被创建，但是实例变量不一定被创建）

# 2、用法

加修饰符 static 即可

# 二、类方法

# 1、实例方法和类方法的区别

1.1 实例方法在创建对象是分配空间，类方法在类加载后就会被分配空间

1.2 类方法不可以操作实例变量 并且 不能调用实例方法（类加载完成后实例变量和实例方法不）

1.3

# 2、用法

加修饰符 static 即可
