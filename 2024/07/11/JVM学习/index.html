# 汇编笔记

  1. 最开始出现的语言是机器语言，由二进制组成 即：01010 由于程序员容易出错还需要记大量的 0101 由此引入了汇编语言指令有两部分组成 操作数 + 操作码
  2. 进制越小，表示相同的数据量，需要的位数就越多
  3. 内存扩张方向，可以自上而下，也可以下而上
  4. 字节序用于规定数据在内存中如何存放，intel 用的小端序
  5. 方法调用等于入栈出栈的过程，满足先进后出
  6. 汇编中用 sp 寄存器和 bp 寄存器用于保存栈底和栈顶
  7. 16 位机器码中 sp 为 16，esp 为 32，rsp 为 64 位

类加载器

  1. 当静态变量为整形时 ，在 clinit 中对其进行初始化，整形值将会放在字节码中， 通过操作数栈来保存，当我们调用 float、double、对象时， 那么将会放入常量池中，在运行时通过 idc 指令，（将他们的地址放入操作数栈中来操作）（切记：是 Idc、iconst、fconst、dconst、bipush 等，都是基于赋值的值的大小来在编译是处理）

# java VS C++

  1. java
    1. GC 处理垃圾
    2. 开发效率高，执行效率低
  2. C++
    1. 手工处理垃圾
    2. 忘记回收
      1. 内存泄漏
    3. 回收多次
      1. 非法访问
    4. 开发效率低，执行效率高

# JVM 内存分代模型（用于分代垃圾回收算法）

  1. 部分垃圾回收器使用的模型
    1. 除 Epsilon ZGC Shenandoah 之外的 GC 都是使用逻辑分代模型
  2. 新生代 + 老年代 + 永久代 (1.7)/ 元数据区（1.8）Metaspace
    1. 永久代 元数据 - class
    2. 永久代必须指定大小限制，元数据可以设置大小，也可以不设置，无上限（受限于物理内存）
    3. 字符串常量 1.7 - 永久代，1.8 - 堆
    4. MethodArea 逻辑概念 - 永久代 - 元数据
  3. 新生代 = Eden + 2 个 suvivor 区

# 清除算法

  • Mark-Sweep（标记清除）
    • 在对象存活较多的情况下效率会比较高
    • 会产生碎片（空间不连续）
    • 需要扫描两遍（1、扫描不可回收对象，2 扫描可回收对象）
  • Copying（拷贝）
    • 在存活对象较少的情况
    • 没有碎片、效率高只扫描一遍
    • 浪费空间
  • Mark-Compact（标记压缩）
    • 扫描两次、需要移动对象，效率偏低

# 触发机制

MinorGC/YGC: 年轻代空间耗尽时 触发

MajorGC/FullGC: 在老年代无法继续分配空间时触发，新生代老年代同时进行回收
