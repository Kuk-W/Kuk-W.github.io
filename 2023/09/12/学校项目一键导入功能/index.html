设计

  1. 如何做到事务回滚
    1. 在进行数据写入操作不要单条数据操作，选择数据批量操作 不仅可以提高速度还可以进行是回滚
    2. 具体逻辑实现：
      1. 将从 excel 扫到的数据放入 map 中并对数据进行处理
      2. 使用 map 中数据 对应的赋值到人的字段上
      3. 批量插入人
    3. 其中 2.1 和 2.2 中有任何错误记录 将记录行号、错误信息、错误原因封装为 map 返回，最后如果发现 dataErrorList…size>0 则不进行数据库操作

遇到的问题

  • 导入速度慢的问题，方法如下
    • 在构建错误消息时，不要使用字符串拼接（ + ），而是使用 StringBuilder 来高效地连接字符串。
    • 当知道验证将会在代码的早期失败时，使用早期返回。使代码更加高效。
    • 每次进行账号验证时都会编译正则表达式模式。由于正则表达式模式是常量，在循环外编译它一次，以避免不必要的开销。

思路一、（所在班级）字段是需要大量查库个写库的操作

  • 扫一次 excel 表格 一条数据提数据去校验是否合法

思路一的问题

  • 单行扫速率极其慢

思路二、

  • 扫两次 excel 表格 第一次针对对数据库操作多的做处理，维护四个 HashSet 分别存放校区、年级、如学年度、班级。避免了大量的查询操作，最后根据 HashSet 去批量创建创建组织
  • 第二次扫 excel 可以直接将学生对应的信息直接 set 到学生字段里 最后进行批量插入人
