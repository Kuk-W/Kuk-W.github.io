RSA简介

是非对称加密算法

优点：

  1. 安全性高：RSA的安全性基于大素数的质因数分解难题，这个难题在目前的计算能力下是不可解的，因此RSA具有较高的安全性。
  2. 公钥易于分发：RSA的公钥可以自由分发，因此在实际应用中，发送方只需要知道接收方的公钥即可进行加密通信。
  3. 数字签名功能：RSA可以用于数字签名，保证数字签名的真实性和完整性。

缺点：

  1. 计算量大：RSA的加密和解密需要进行大量的数学运算，因此计算量比较大，特别是在处理大数据时，会消耗大量的时间和计算资源。
  2. 密钥管理困难：RSA需要管理一对密钥，其中私钥必须保密，如果私钥泄露，就会导致通信的安全性受到威胁。
  3. 无法加密超过密钥长度的数据：RSA的加密过程中，只能加密比密钥长度小的数据，如果要加密长数据，需要进行数据分块和多次加密，这会增加计算量和通信开销。

对于数据字节长度过长的不太友好 费时

AES简介

对称加密

aes使用时会每次都传输新的秘钥

优点：

  1. 安全性高：AES提供了高强度的加密，可以有效地保护敏感数据。它使用块密码和分组密码的结合方式，抵御了许多攻击，特别是侧信道攻击和差分攻击。
  2. 速度快：AES的加密和解密速度非常快，适合用于处理大量数据的场景。AES在硬件和软件中都有高效的实现，使得它成为一种流行的加密算法。
  3. 算法公开：AES的算法公开，任何人都可以查看和分析其加密过程，这有助于提高其安全性。

缺点：

  1. 密钥管理困难：AES是一种对称加密算法，密钥需要在发送方和接收方之间进行传递，因此需要进行安全的密钥管理。
  2. 无法防止重放攻击：由于AES只提供加密和解密功能，而无法防止重放攻击，因此在应用中需要采取其他措施来防范重放攻击。
  3. 无法进行数字签名：AES是一种对称加密算法，无法用于数字签名和认证，这需要使用其他加密算法来实现。

RSA+AES实现

为什么使用RSA + AES混合加密

1.加密介绍

RSA加密: 属于非对称加密，公钥用于对数据进行加密，私钥对数据进行解密，两者不可逆。公钥和私钥是同时生成的，且一一对应。比如：客户端拥有公钥，服务端拥有公钥和私钥。客户端将数据通过公钥进行加密后，发送密文给服务端，服务端可以通过私钥和公钥进行解密。
AES加密: 属于对称加密，简单点说就是，客户端用密码对数据进行AES加密后，服务端用同样的密码对密文进行AES解密。
2.加密思路

利用 RSA 来加密传输 AES的密钥，用 AES的密钥 来加密数据。
既利用了 RSA 的灵活性，可以随时改动 AES 的密钥；又利用了 AES 的高效性，可以高效传输数据。
3.混合加密原因

单纯的使用 RSA（非对称加密）方式，效率会很低，因为非对称加密解密方式虽然很保险，但是过程复杂，耗费时间长，性能不高;
RSA 优势在于数据传输安全，且对于几个字节的数据，加密和解密时间基本可以忽略，所以用它非常适合加密 AES 秘钥（一般16个字节）;
单纯的使用 AES（对称加密）方式的话，非常不安全。这种方式使用的密钥是一个固定的密钥，客户端和服务端是一样的，一旦密钥被人获取，那么，我们所发的每一条数据都会被都对方破解;
AES有个很大的优点，那就是加密解密效率很高，而我们传输正文数据时，正好需要这种加解密效率高的，所以这种方式适合用于传输量大的数据内容;
基于以上特点，就有了我们混合加密的思路
