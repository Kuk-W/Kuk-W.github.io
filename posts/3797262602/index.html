计算机网络基础

  • 计算机之间的通信 ：主机A的某个进程和主机B的某个进程之间进行通信
    • C/S（client、server）

计算机网络分类

按照作用范围划分
  • 广域网 也叫远程网
    • 作用范围：几十到几千公里
  • 地域网
    • 作用范围：5~50公里 一般一个城市的范围
  • 局域网
    • 类似一个网吧
  • 个人区域网PAN
    • 手机和电脑通过蓝牙连接
按照线路接结构划分
  • 星型（也称拓扑结构）在局域网中使用较多
    • 优点
    • 缺点
  • 总线型
    • 材料：同轴电缆、光缆
    • 通知方式：广播式
    • 优点
      • 结构简单
      • 走线量小
      • 成本低
      • 扩充灵活
    • 缺点
      • 故障诊断和隔离比较困难
  • 环形（令牌环网）
    • 令牌：一组特殊格式的帧
    • 材料：电缆
    • 优点
      • 布线简单
    • 缺点
      • 效率低
  • 网状型
    • 优点
      • 可靠性高
      • 效率高
      • 延迟小
    • 缺点
      • 控制复杂、软件复杂
      • 线路费用高、不以扩充

计算机网络的性能指标

  • 速率
    • 1B = 8b B：Byte 字节，b：bit 位
    • 100(bit/s)
    • 表示大小时 1MB = 2^10*8 bit
    • 表示速率时 1MB/s = 10^3 bit/s （又叫数据率或者）
  • 带宽
    • 用来表示网络中某通道的输出传输能力 ==最高==数据率
    • 单位b/s
    • 宽带100兆：出口网速是100Mb/s
  • 吞吐量
    • 表示在单位时间内某个通道通过的数据量 表示==实际==数据率
  • 时延
    • 数据从一端发向另一端所需要的时间
    • 发送时延
      • 主机或路由器发送数据帧所需要的时间
      • 发送公式 ： 发送时延 = 数据帧长度（bit）/发送速率（bit/s）
    • 传播时延
      • 电磁波在信道中传播一定的距离所需要花费的时间
      • 公式：传播时延 = 信道长度（m）/电磁波在信道上传输的速率（m/s）
      • 自由空间 ：3* 10^8 m/s
      • 铜线：2.3* 10^8 m/s
      • 光纤：2* 10^8 m/s
  • 往返时间
    • 表示发送端将数据发送到传输媒介开始， 到 发送端接收到接收端确认 的总共经历时间(不包含发送延时)
    • RRT：包括三部分 链路传播时间（固定）+末端系处理时间（固定）+路由器缓存排队时间和处理时间
  • 利用率
    • 分为网络利用率和信道利用率两种
    • 信道利用率：表示某信道有百分之几的时间是被利用的（有数据通过的时候）
    • 网络利用率：全网络的信道利用率的加权平均值
    • 排队论：当某信道的利用率增大时，改信道的==时延==会增大
    • 公式：D = D0/1-U
      • 其中D表示当前网络的时延
      • D0表示网络空闲的时延
      • U表示网络的利用率
  • 丢包率
    • 又叫分组丢失率，在一定时间范围内，传输过程中丢失的分组数量与总分组数量的比率
    • 分组丢失的情况
      • 分组在传输过程中出现==误码==，被节点丢弃。
      • 分组到达一个 分组已满 的交换主时被丢弃，在通信较大是会造成网络拥塞

物理层

主要材质有：电缆、光纤

在OSI参考模型中，物理层是参考模型的最低层，也是OSI模型的第一层。==物理层的主要功能是==：利用传输介质为数据链路层提供物理连接，实现比特流的透明传输。物理层的作用是实现相邻计算机节点之间比特流的透明传送，尽可能屏蔽掉具体传输介质和物理设备的差异，使其上面的数据链路层不必考虑网络的具体传输介质是什么。

数据链路层

数据链路层会封装好MAC帧，将其交付给物理层

数据链路层（Data Link Layer）是OSI模型的第二层，负责建立和管理节点间的链路。在计算机网络中由于各种干扰的存在，导致物理链路是不可靠的。因此这一层的==主要功能是==：在物理层提供的比特流的基础上，通过差错控制、流量控制方法，使有差错的物理线路变为无差错的数据链路，即提供可靠的通过物理介质传输数据的方法。

网络层

网络层（Network Layer）是OSI模型的第三层，它是OSI参考模型中最复杂的一层，也是通信子网的最高一层，它在下两层的基础上向资源子网提供服务。==其主要功能是==：在数据链路层提供的两个相邻端点之间的数据帧的传送功能上，进一步管理网络中的数据通信，控制数据链路层与传输层之间的信息转发，建立、维持和终止网络的连接，将数据设法从源端经过若干个中间节点传送到目的端（点到点），从而向传输层提供最基本的端到端的数据传输服务。具体地说，数据链路层的数据在这一层被转换为数据包，然后通过路径选择、分段组合、顺序、进/出路由等控制，将信息从一个网络设备传送到另一个网络设备。数据链路层和网络层的区别为：数据链路层的目的是解决同一网络内节点之间的通信，而网络层主要解决不同子网间的通信。

传输层

OSI下3层的任务是数据通信，上3层的任务是数据处理。而传输层（Transport Layer）是OSI模型的第4层。该层提供建立、维护和拆除传输连接的功能，起到承上启下的作用。该层的==主要功能是==：向用户提供可靠的端到端的差错和流量控制，保证报文的正确传输，同时向高层屏蔽下层数据通信的细节，即向用户透明地传送报文。

会话层

会话层是OSI模型的第5层，是用户应用程序和网络之间的接口，==该层的主要功能是==：组织和协调两个会话进程之间的通信 ，并对数据交换进行管理。当建立会话时，用户必须提供他们想要连接的远程地址。而这些地址与MAC地址或网络层的逻辑地址不同，它们是为用户专门设计的，更便于用户记忆。域名就是一种网络上使用的远程地址。==具体功能如下：==

  1. 会话管理：允许用户在两个实体设备之间建立、维持和终止会话，并支持它们之间的数据交换。
  2. 会话流量控制：提供会话流量控制和交叉会话功能。
  3. 寻址：使用远程地址建立会话连接。
  4. 出错控制：从逻辑上讲会话层主要负责数据交换的建立、保持和终止，但实际的工作却是接收来自传输层的数据，并负责纠正错误。
表示层

表示层是OSI模型的第六层，它对来自应用层的命令和数据进行解释，对各种语法赋予相应的含义，并按照一定的格式传送给会话层。==该层的主要功能是==：处理用户信息的表示问题，如编码、数据格式转换和加密解密等。==具体功能如下：==

  1. 数据格式处理：协商和建立数据交换的格式，解决各应用程序之间在数据格式表示上的差异。
  2. 数据的编码：处理字符集和数字的转换。
  3. 压缩和解压缩：为了减少数据的传输量，这一层还负责数据的压缩与恢复。
  4. 数据的加密和解密：可以提高网络的安全性。

应用层是OSI参考模型的最高层，它是计算机用户，以及各种应用程序和网络之间的接口，==该层的主要功能是：==直接向用户提供服务，完成用户希望在网络上完成的各种工作。它在其他6层工作的基础上，负责完成网络中应用程序与网络操作系统之间的联系，建立与结束使用者之间的联系，并完成网络用户提出的各种网络服务及应用所需的监督、管理和服务等各种协议。此外该层还负责协调各个应用程序间的工作。==应用层的具体功能如下：==

  1. 用户接口：应用层是用户与网络，以及应用程序与网络间的直接接口，使得用户能够与网络进行交互式联系。
  2. 实现各种服务：该层具有的各种应用程序可以完成和实现用户请求的各种服务。

OSI七层模型总结
应用层：产生网络流量的程序

表示层：传输之前是否进行加密或者压缩处理

会话层：查看会话，查木马 netstat-n

传输层：可靠传输、流量控制、不可靠传输

网络层：负责选择最佳路径、规划ip地址

数据链路层：帧的开始和结束、透明传输、差错校验

物理层：接口标准、电器标准、如何更快传输数据

传输层：tcp协议、udp协议、TLS./SSL、SCTP等

网络层：ip协议（IPV4和IPV6）、ICMP协议、IPSce协议

主要解决五个问题

1、报文拆分

2、增加协议头

3、解决相邻接点之间的传输

4、解决路由和寻址问题

5、数据重组

TCP协议

def：TCP全名是(Transport Control Protocol)，是一个可以提供可靠的、支持全双工、连接导向的协议，因此在客户端和服务端之间传输数据的时候，是必须先建立一个连接的

  • 连接
    • 是虚拟、抽象的概念
    • 也会被成为会话
    • 使用通信更稳定、安全
  • 双工
    • 单工：任何时刻数据只能单向发送
    • 半双工：允许数据在两个方向上传输，在某时刻只允许数据在一个方向上传输
    • 全双工：任何时刻都能双向发送数据
  • 可靠
    • 无损传输：多播、无序数据恢复有序
三次握手四次挥手

https://blog.csdn.net/weixin_45393094/article/details/104965561

为什么握手是三次 挥手是四次

断开连接的过程中 客户端发送FIN后 服务端没有办法直接将FIN和ACK包同时返回给客户端 （因为此时服务端可能之前的 资源还没有释放、数据还没处理完毕）

传输数据的过程

  • TCP协议

    • 报文拆分
      • 应用层数据很大无法一次传输完成
      • 拆分完后可实现并行传输
    • 增加TCP头
    • 数据重组

  • 顺序保证

    • TCP序号：发送序号Seq、接受序号Ack

      • 一端的发送序号是另一段的接收序号

    • 时间窗口

      • 在改时间段内发送的数据完整的话接受否则抛弃该时间窗口的所有有数据

    • TCP头

      • 源端口：描述发送方机器上的应用
      • 目标端口：描述接收方服务器上的应用
      • 发送序号seq和接受序号ack

    • TCP标志位

      • ECN：显示拥塞协议，有助于帮助解决延迟和丢包问题
      • URG：紧急标志位（置1 会被优先执行）
      • SYN：同步序号
      • FIN：终止连接
      • ACK：响应（确认包）
      • PSH：传送数据
      • RST：重置连接

  • TCP报文

    • 校验和（16位）
      • 纠错能力：保证数据的可靠性
      • 16位的Checksum字段
        • 是一个函数，可以把原文映射到一个不可逆的16bit 编码（可以知道传输过程中有没有发生变化）
      • 流控能力：控制发送速率（利用滑动窗口）

TCP协议原理

  1. 可靠的传输机制
    1. 确认应答机制
    2. 超时重传机制
    3. 连接管理机制
    4. 流量控制
    5. 拥塞控制
  2. 效率机制
    1. 滑动窗口
    2. 延迟应答
    3. 捎带应答
TCP协议特点总结
  • 有连接：通过三次握手建立连接后才可接发数据，TCP协议是全双工的，即每端既可以发也可以收
  • 可靠传输：网络数据传输是一跳一跳的，经过路途中的设备可能发生数据丢失，可靠传输是可能发生数据丢失但有机制保证对方能接收到
  • 面向字节流：可以多次的收发数据（连接没有关闭时，可以多次的接收和发送数据）
  • 有接收缓冲区和发送缓冲区：发送数据时，是先写到发送缓冲区，再刷新缓冲区（flush）
  • 大小不受限制：多次的收发数据，每次的数据可以很大

IPV4和IPV6区别

  • 地址
    • 地址数量：IPV4有4个8位 共32位 IPV6有8个16位，共128位
    • 分隔符号：
      • IPV4:：例如：192.168.2.22用.去分隔用0-255去表示
      • IPV6：例如:0123:4567:89ab:cdef:0123:4567:89ab:cdef用，分隔 每个是一个16位的16进制数字 。四个字符
    • 书写方式：
    • 寻址
      • 全局单播
        • 站点前缀：48位，一般由ISP分发运营商
        • 子网号：16bit用于站点内部区分子网
        • 接口号：64位用于站点内部区分设备
      • 本地单播
      • 分组多播
